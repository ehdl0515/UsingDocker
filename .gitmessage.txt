# [타입] 리스트
#   feat    : 기능 (새로운 기능, 기능내용변경)
#   refactor: 프로덕션 코드 리팩토링, 새로운 기능이나 버그 수정없이 현재 구현을 개선한 경우
#   fix     : 버그 (버그 수정)
#   version : 버전 (버전 변경)
#   style   : 스타일 (코드 형식/로그/오타 수정, 주석추가, 변수명변경 등: 비즈니스 로직에 변경 없음)
#   docs    : 문서 (문서 추가, 수정, 삭제)
#   test    : 테스트 (테스트 코드 추가, 수정, 삭제: 비즈니스 로직에 변경 없음)
